O que é um arquivo .ejs? (Modelo JavaScript incorporado)

Um arquivo EJS contém código escrito na linguagem templating JavaScript (EJS) incorporada, que é utilizada para gerar marcação HTML usando JavaScript. Ele é normalmente usado como parte de um aplicativo web e inclui tags que o mecanismo EJS substitui por informações de um banco de dados para produzir um tempo de execução.

Você provavelmente só encontrará um arquivo EJS se você for um desenvolvedor web utilizando o mecanismo de templating EJS para um site dinâmico. O mecanismo templating pode ser utilizado no frontend (executado em navegadores da Web) ou no backend (executado com NodeJS, um ambiente de tempo de execução JavaScript) de sites. Alguns sites dinâmicos que podem incorporar o EJS incluem lojas online ou diretórios que armazenam informações de perfil para pessoas.

Um modelo EJS normalmente contém elementos estáticos que compõem a estrutura da página, juntamente com um ou mais elementos dinâmicos. Os elementos dinâmicos serão preenchidos com dados de uma fonte externa, como um banco de dados, em um momento posterior em que o usuário solicita a página web.

Por exemplo, um modelo EJS pode ser usado para várias páginas da web do produto O modelo contém elementos estáticos para o layout da página que tornam a aparência das páginas do produto semelhante. No entanto, ele também armazena elementos dinâmicos ("productname" e "productdescription" tags neste exemplo) que serão preenchidos com o nome do produto e a descrição de um banco de dados quando um usuário solicitar a página.

Você pode abrir arquivos EJS com qualquer editor de texto, uma vez que os arquivos são salvos em formato de texto simples. No entanto, você pode querer usar um editor de código-fonte que suporte o idioma ou permite que você instale um plugin que suporte o idioma para fornecer recursos de codificação úteis.

NOTA: O EJS é um dos vários motores de templating JavaScript usados por desenvolvedores web, juntamente com Pug (ex-Jade), Bigode, Guidão, HAML, Vash e Nunjucks.
