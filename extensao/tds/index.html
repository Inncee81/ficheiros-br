O que é um arquivo .tds? (Modelo de designer do Adobe LiveCyle)

Um arquivo TDS contém um modelo criado pelo Adobe LiveCycle Designer, uma ferramenta de criação de formulários incluída no Adobe LiveCycle Enterprise Suite (ES). Ele contém a estrutura de um modelo de formulário que é usado para automatizar a reprodução de formas. Os arquivos TDS incluem os componentes e configurações do modelo, como layout da página e informações de formatação, scripts, imagens e fontes.

Mais Informações

O Adobe LiveCycle ES é uma plataforma usada por empresas e agências governamentais para melhorar a correspondência com os clientes. LiveCycle Designer é uma ferramenta incluída no LiveCycle ES que é usado para projetar documentos e formulários para capturar e processar dados dos clientes. Esses formulários e documentos são mesclados com dados de .XML corporativos para gerar documentos e formulários personalizados e interativos que podem ser distribuídos de várias maneiras, como arquivos .PDF, HTML5 e impressão.

Os arquivos TDS são criados para armazenar modelos que podem ser usados para a reprodução de formulários e documentos do Adobe LiveCycle ES. Você pode importar um arquivo TDS para Designer selecionando File → Open ou arrastando e soltando o arquivo TDS no espaço de trabalho Designer.

NOTA: Enquanto o Adobe LiveCycle Designer é normalmente usado como uma ferramenta incluída no Adobe LiveCycle ES, ele também está incorporado no LiveCycle Workbench ES4 e disponível como um programa autônomo.

O que é um arquivo .tds? (Arquivo de símbolos de depuração turbo)

Contém uma tabela de símbolos usados pelo programa Borland Turbo Debugger; criado pelo linker Borland e inclui dados referenciados durante o processo de depuração de software.

Mais Informações

Turbo Debugger está incluído no Borland C++ Builder, bem como em programas anteriores de desenvolvimento de software Borland.
